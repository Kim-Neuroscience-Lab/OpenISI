
# OpenISI — UI/UX Design System

## Design Philosophy

### Core Principles

| Principle | Implementation |
|-----------|----------------|
| **Confidence over speed** | Never let user start acquisition without validation |
| **State is always visible** | User always knows: Where am I? What's happening? What's next? |
| **Smart defaults, full control** | Works out of box, everything overridable |
| **Fail gracefully** | Errors are recoverable, warnings are actionable |
| **Dark room first** | Low brightness, high contrast, no surprises |

### User Mental Model

```
"I want to run an ISI session. The software should:
 1. Know my rig (detect hardware, remember settings)
 2. Let me verify everything works (test/preview)
 3. Guide me through setup (focus, confirm)
 4. Run reliably (clear progress, no surprises)
 5. Tell me it worked (QC, results)"
```

## Summary

This design system provides:

1. **Clear phase progression** — Always know where you are
2. **Smart defaults** — Works immediately for common cases
3. **Full control** — Everything overridable when needed
4. **Confidence building** — Test before committing
5. **Dark room friendly** — Won't blind you or the animal
6. **Fail gracefully** — Errors are clear and recoverable
7. **Scientific rigor** — QC metrics throughout
